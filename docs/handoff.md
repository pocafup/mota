### §S7（本 session·2026-06-13）GA 第三棒：fitness 终局局面价值【已落地】
> 本棒**只**实现 fitness + 双权重标定 + 单测，**未碰 GA 种群/选择/交叉变异/主循环**。已 commit（`solver/fitness.py` + `tests/test_fitness.py` + `analysis/calibrate_fitness.py` + `analysis/probe_key_reachability.py`，全套 **164 pytest 绿**）。

- **契约（`ga_design.md` 钉死点3·κ=1 守死）**：fitness **只对一条已跑完死路线的终态评一次分**，绝不参与中途逐步决策、绝不反馈 decoder/navigate_to。潜力项都是这条路线**真实留下的既成事实**（地上血瓶/地上钥匙），不是"守着没拿的期权"→ 结构上不构成 κ=1。
- **公式**：`fitness(final) = 主干 equiv_hp_over_roster(一区 roster) + w_potion·一区地上剩余血瓶 + 钥匙家底 + win_bonus(通关)`；死亡 → `DEATH_FLOOR(−1e9) + 深度`（恒 ≪ 任何活态，给"虽死但更深"爬坡梯度）。**钥匙家底 = w_key·手里余钥匙(已兑现满权重) + Σ_地上够得到钥匙 max(0, w_key−守怪损血)**。
- **钥匙口径（A类不碰 door_value B）**：地上钥匙=这条死路线终态客观还剩的家底（对齐血瓶），按守怪损血**扣血成本**防 κ=1 裸计数（"13 血净赚 1 把"经济学）。**可达=预算门控**：楼梯出发能走过【终态手里有该色钥匙的门】才算够得到（乐观近似：1 把当整色门可过，因钥匙便宜·门→钥匙链自给）；没钥匙的门当墙、守怪打不动断边。可达绑**终态手里钥匙预算**=既成事实(type-A)，**不是**把钥匙按未来开门折算(type-B 第二阶段红钥长臂，第一版不碰)。数据坐实见 `analysis/probe_key_reachability.py`。
- **标定（真对照 cap480k 718耗尽 vs 689高潜力）**：`w_potion=1.5 ∈[1.0,2.0]`、`w_key=39 ∈[26,52]`，**都取稳健区间中值**（非 689 险胜临界：w_potion 反超临界 0.40、w_key break-even 13，都远低于选定值）。分项对账：主干 689 输 **−210**（更瘦→压制弱）、血瓶项 **+375**（地上多 250 血瓶）、钥匙地上项 **+95**（MT4 三把 13 血黄门钥匙·718 已吃掉/689 留地上）→ **净 +260 反超**。**689 反超靠血瓶+钥匙双潜力一起补主干劣势**，缺一不可、每项物理意义干净。
- **单测**（`tests/test_fitness.py` 10 个）：fitness(689)>fitness(718)、死亡态≪任何活态、更深死>更浅死、通关恒+win_bonus、双潜力反超、血瓶线性、钥匙分解、便宜算/贵归 0、`test_key_budget_gating`（清空手里钥匙→黄门当墙→够得到钥匙严格变少＝可达绑预算的硬证）、total 对账。
- **红线守住**：只动 fitness，**未碰 GA 主循环**；fitness 只终评不进搜索剪枝。
- **⚠ 待办（落盘别做·GA 主循环验证后的校准项）**：玩家通关存档（根目录·首次到 MT10 在 **tok788**）可作 fitness 的**第二道独立校准**——但需先**对齐截断口径**（玩家 tok788=刚进 MT10 vs beam 718/689=MT10 内自然收敛，进度不同**不可直接比**）。最佳时机=最小 GA 跑出一区路线后，**三方对照**（GA 路线 / 689 / 玩家 tok788）看 fitness 排序是否合理、GA 是否逼近人类打法。

### §S8（本 session·2026-06-13）GA 第四棒：decode 最小闭环【已落地】+ navigate_to 吸收语义缺陷·路2 决策
> 本棒**只**实现 decode 闭环 + 单测 + 诊断脚本，**未碰 GA 种群/选择/交叉变异/主循环**。已 commit（`extract/ga_decode.py` + `tests/test_ga_decode.py` + 诊断 `extract/ga_decode_diag.py`，全套 **174 pytest 绿**）。

- **闭环成果**：`decode(chromosome, start) -> (tokens, final)` 串 navigate_to（依次定向导航各目标、够不到跳过、死亡/通关停）+ fitness 终评。闭环门 **A[剑,盾] ≥ B[剑,小宝石,盾]** 复现 fitness 对 **689>718**「留潜力 > 耗资源」机制（A 靠血瓶+HP 潜力赢、B 属性高但 HP 低）。
- **⚠ 诊断发现 C[盾,剑]==A 非良性（撤回"符合预期"）**：`extract/ga_decode_diag.py` dump 坐实——`navigate_to(start, 盾)` 这条腿步数=241、终态 ATK 10→21（顺路抓了剑）、与 A 完整 241 步逐 token 相同；物理顺序也是先剑(第81步)后盾(第240步)。排除缓存误命中（cache=None 与带 cache 逐字段同）、排除 decode 重排（leg0=盾 leg1=剑 忠实按序）、排除静默跳过（leg0 reached=True）。
- **缺陷根因（dump 坐实非推断）**：navigate_to **进块即贪婪 `_absorb` 吸光所有零损血道具**。去盾的最优路顺路经过剑格 → 吸剑 → `[盾,剑]` 基因物理上变成 `[剑,盾]`、与 A 逐 token 相同。**decode 没错、navigate_to 符合自身契约**，错在【基因表示与贪婪吸收的接缝】：navigate_to 在"导航"名义下替 GA 做了"拿不拿"的策略决策、**越权**。
- **为什么必须修（玩家游戏知识坐实）**：MT4 那 3 把钥匙很优秀、在"开局→盾"主干路影响范围内（玩家最优解**盾前**取 MT4、而 689/718 是**盾后**取——"何时取 MT4"是真实策略维度、正是 GA 该搜的）。navigate_to 顺路把钥匙吸了 → GA **永远无法表达**"先拿盾、留 MT4 钥匙当潜力"这 689 式策略 → 在 GA 最该发挥处失去控制权。**冲突本质：navigate_to 无差别贪婪吸收 ⊥ fitness"留潜力"理念**。

#### 【二、V_boss/fitness 关系厘清（收回一个过度简化判断·见 memory `feedback-vboss-fitness-relationship`）】
- **收回"HP155 是 fitness 判不准实锤"**：fitness 评的是"地上剩余血瓶 + HP + 钥匙的**等价 HP**"（不只瞬时 HP）。HP155 低但地上有血瓶（钥匙能拿）→ 等价 HP 不低 → fitness 给那个分**合理**、是 GA 当前 navigate 局限下的**合理最优**，非废解。
- **★V_boss 该评 SEAM（boss 入口到达态）、不是中间态瞬时**：到达态已整合早期所有决策 + 路上回血 → 在 seam 查**自动满足"前期烧血没关系"**（按净到达态判·阈值下 V_boss=0 不是负）。在中间态查瞬时属性会**前期误罚烧血**（漏掉中间→seam 的回补）。§S27"逐状态查 V_boss 当稠密奖励"的提法因此有缺陷——稠密是在 (a,d,h)-到达空间里、用在 seam 不是每个 timestep。
- **★配合非替换**：fitness 管中间态潜力（含可达血瓶/钥匙的线性等价 HP·处处稠密梯度）；V_boss 管 seam 终值（boss 段 nonlinear 拐点 + 连杀 sequencing + key-gating·线性等价 HP 漏掉的）。**score = fitness + λ·V_boss**（V_boss 作非负 terminal bonus 叠在 fitness 上）→ 既解死区无梯度、又不前期误罚烧血。**别简单"换掉 fitness"**（换掉抹掉 fitness 信号、前期误罚烧血）。
