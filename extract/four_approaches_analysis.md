- `seg_identify_zone1.classify` L74-75：钥匙门 = `("pay","door")`；`vzone._passable` 只挡 `"wall"`；
  `vzone._enter_cost` 只对普通怪(mon_cache)收 toll。
- ⇒ **`build_zone` 跨层图里 门/道具/NPC/special怪/事件格 全部可穿、零代价**，只有墙 + 普通怪挡路。
- ⇒ `_toll_dist_from`(pull 距离) 和 `shortest_toll`(区势能 D) **早已是"跨层、穿门、看得到门后/远处
  目标格级距离"**。handoff 方向一(2026-06-11)写的"pull 距离困在当前层+楼梯算子链"**对当前代码已不准确**。

**真正的分裂 = 两套几何**：

| | 打分几何（vzone 跨层图 + 区势能） | 搜索展开几何（quotient） |
|---|---|---|
| 跨层 | ✅ 楼梯免费边、跨层 Dijkstra | 楼梯 = 逐个 `stair` 算子，一个一个 wave 展开 |
| 门 | ✅ 零代价穿（乐观），**看得到门后** | `door` 算子，要 `keys>0` 才开，没钥匙=死边界 |
| 身份 | 无（纯距离/势能） | `_qfp` 含 `current_floor`，**每层独立块** |
| 吸收 | — | `_absorb` 只吸**当前单层**自由块内 |

**延迟满足的谷发生在展开层**：打分层 G 在崖顶给满额、距离也跨层看得到崖顶，但穿谷中间态(HP 低、G 未
兑现)在 beam wave 截断时被高 HP 兄弟按标量 V 压掉。**"看得到"(打分) ≠ "走得到"(展开)。**

## 2. 思路四是不是前提 —— 修正判断

问："三条思路是不是都要先看到门后/跨块的崖顶?" → **不是。**

破谷所需的"崖顶定位 + 距离"，打分层（`_toll_dist_from` 跨层穿门）**已经能给** → 一/三**不以联通块视野
为前提**就能落地。

联通块视野**真正是前提**的只有三件，且都不是"破谷"：
1. **钥匙价值**（把门后回报算进"花钥匙开门"决策）；
2. **飞行可达**（30 飞行楼传补飞行边）；
3. **展开层压平楼梯虚假深度**（能塌缩纯走廊/楼梯步数；但 idea2 谷是真代价 3 怪 2 门堆叠，联通块**压不掉**）。

> **结论：handoff 把"联通块视野 = 破谷前提"高估了。** 破谷靠一/三即可、距离几何已就位；联通块视野是
> **钥匙价值/飞行**的前提，与破谷**并行、不互为前置**。

**caveat（诚实）**：区势能 `_future_potential` 看的是"区内非当前层存活怪 toll" = 区级方向、**不定位具体
门后道具**；定位门后那颗宝石距离的是 `_toll_dist_from` = 格级。approach 梯度要用后者。

## 3. 四条能否配合 + 推荐

- **一+三天然互补**：三做护栏（谷底态不被高 HP 兄弟挤掉）、一做信号（谷底正梯度），双保险破谷；三的
  "不进价值"兜住一的 κ=1 风险。
- **二排除**（κ=1）。
- **四降级**为并行的"钥匙价值/飞行"专项，不挡破谷。

**建议下一步**：三打底（最安全、不碰价值）→ 叠一（满额兑现 G 压就近病）→ 在 idea2 MT1 谷上受控验证
（谷-崖是现成靶心、谷底 Δ 排序键已量化）。

## 4. 联通块视野怎么落地 + 和现有 quotient 块/吸收视野关系

落地 = 把**展开几何**从单层升级到跨层（打分几何已跨层、不用动）：
- **(a)** `_free_cells` 单层 floodfill → 跨 `zone["links"]` 的**多层 floodfill**（楼梯格跳对面继续）；
- **(b)** 楼梯不再当 `_boundary_ops` 的 `stair` 算子/不打分，并进块内零代价移动；`_qfp` 不按 `current_floor` 切；
- **(c)** 距离用块内真实跨层最短路（`_toll_dist_from` **已就位**）；
- **(d)** `_absorb` 跨块吸收；门后用**乐观并块**预览（门当可穿，**仅用于打分/目标定位**，真实展开仍要钥匙）。

**和 quotient 块关系 = 把单层版升级成跨层版。** 但这是**大手术**：`_qfp` 指纹空间、`visited` 无损去重、
`beam` 分坑维全受牵动，**难保零回归、违反"一次只动一个变量"**。

> **建议：先不动 `search_quotient` 展开几何（高风险）**，先在**打分/引导层**用现成跨层距离做钥匙价值
> （§5），验证收益再评估是否动展开层。

## 5. 钥匙价值怎么和联通块视野配合

- **联通块视野（乐观并块、门当可穿）→ 看见门后那片有什么**（道具/钥匙/更高区势能）；
- **钥匙价值 = 给"花钥匙开门"一个回报信用** = 门后那片（乐观并块）价值，进 beam 排序键（**不进
  value_vector**，守红线）；
- **拿到才兑现**：门后价值在"穿过门 + 吸到门后道具"那刻满额兑现；开门那刻只给 approach 引导（像 pull）。
  否则"守着门不开也算门后潜力" = κ=1；
- 联通块视野给"门后有什么"**几何**，钥匙价值给"值不值得花钥匙"**估值** = idea3 §4B"三位一体"那个开门
  动作的解。

## 7. 思路三命门：分坑 key 怎么界定"穿谷者"（纯护栏·不进价值·不退化成 approach）

**结论：分坑维 = "通向高净值目标的必经关口里、已跨过哪些"**（门已开 + 怪已杀的 frozenset），与现有
`_stairs_key` 复合。**不是距离、不是价值。**

**为什么不用"`_toll_dist_from` 距离在减小"（玩家问的那个）**：
- 距离是连续信号 = approach 形态 → 用作分坑就**退化成思路一**（玩家划的红线）。
- 距离**可逆**（退一步 dist 就涨）→ 不反映"已付出的不可逆投资"。护栏要保护的恰是"已花钥匙开门、已损血
  杀怪"这种**沉没承诺**的态——分坑维该按这笔不可逆投资分，不按当前位置分。

**具体形态（推荐）**：
- 预计算（驱动层·一次）：目标集 `T` = 本区 top-K 高 ΔRP₀ 未兑现宝石（ΔRP₀ 直取 `detect_big_items` 的
  `ranked`，现成）。对每个 `t∈T` 用 toll-Dijkstra 算 entry→t 最省损血路，收集路上**关口** = 门格
  (DOOR_KEY_MAP) + 普通怪格 (mon_cache)。`G_all` = 各路关口并集。
- 分坑维 `valley_key(s)` = `frozenset{ g∈G_all : g 在 s 已跨过(门已开/怪已杀，读 state flag/entities) }`。
- 复合：`diversity = (_stairs_key(s), valley_key(s))`。

**为什么有界**（守 `_stairs_key` 同条"永不退化每态一坑"）：关口按路序跨越、已实现子集≈路径前缀 →
实际坑数 O(|G_all|) 非 2^|G_all|；region-local + top-K(如 K=2) → |G_all| 小（一个区必经门/怪约 5-10 格）。

**为什么是纯护栏**：`valley_key` 只进 `diversity_key_fn` 分区、绝不进 score。穿谷态因自成坑、坑内 Pareto
骨架非支配而**存活**；它的分仍是诚实的低 V。**不改价值、不犯 κ=1、不犯就近病。**

**为什么识别"朝宝石走"而非"瞎掉血"**：关口专指【通向 T 的必经路】上的门/怪。在别处打无关怪掉血 = 不碰
`G_all` = `valley_key` 不变 = **不进新坑**。只有跨 gem-path 关口才换坑。

**棘轮机制（三为什么单独可能就破谷）**：开门1→`valley_key` 变（进近空坑）→骨架保住→杀怪1→再换更深坑
→…→踏宝石（崖）。quotient 的 door/kill 算子在每关口生成下一坑子态，分坑逐坑保护把态**一路抬过谷**。

**诚实 caveat**：三只保证穿谷态【不被剪】、不主动【偏好】扩它（quotient 本枚举全部边界算子、扩展非选择性
→大概率够）。但若坑内多穿谷态、骨架只留 V 最高的，可能留了个乱走的 → 这是叠一（approach 让"朝宝石"那个
V 最高）的用处。**需小扩展**：`_toll_dist_from` 加前驱 map（或单独一遍带 pred 的 Dijkstra）重建关口路径，
预计算级、不进热路径。

## 8. 思路一命门：approach 梯度怎么靠满额兑现 G 压住就近病

**结论：思路一 = 把 pull 引导从 `big_cells` 扩到 `T`（高 ΔRP 小宝石）。就近病的压制不需新证明——复用本
session G 的命门④-④ 结构不等式即可。**

**机制（复用现成机器）**：
- approach 项：`β_a·Σ_{t∈T·未兑现·够得到} ΔRP(t,当前)/(1+dist)`，dist 用 `_toll_dist_from`（已跨层穿门）。
  = `pull_big` 把 `big_cells` 换成 `T`、机器一模一样。
- G（拿取奖励·本 session 07bd423 已落）：小宝石拿到给满额 `β_small·ΔRP₀`，已 wired（`build_pickup_bonus`
  的 `beta_small`）。
- `beam_score_extra(s) = β_big·pull_big(big_cells) + G(s) + β_a·pull(T)`。

**就近病为什么被结构性压住（= 命门④-④ 同一不等式）**：
- 守着 dist=1 旁边不拿：`approach = β_a·ΔRP(当前)/2`。
- 踏上去拿：`G = β_small·ΔRP₀`，且该 t 的 approach 归 0（离场）。
- `ΔRP₀(参照态·最弱) ≥ ΔRP(当前)`（边际减伤递减、ref 最弱=上界，`build_pickup_bonus` 已 assert、guard 钉死）。
- ⇒ 只要 **`β_a ≤ β_small`**：`拿 = β_small·ΔRP₀ ≥ β_a·ΔRP₀ ≥ β_a·ΔRP(当前) > β_a·ΔRP(当前)/(1+dist)|_{dist≥1} = 守着`。
- 拿严格 > 守着 → approach 斜坡**没有低于"拿到"的局部极大** → 无悬停不动点 → **就近病结构性消除**。

**唯一新约束：`β_a ≤ β_small`（approach 系数 ≤ 兑现系数）。这一条就够。**

**多颗小宝石互相干扰**：用 SUM（与 `pull_big` 一致）即安全——每颗 t 的【拿 t】=`β_small·ΔRP₀(t)` ≥【t 的
approach 贡献】（per-gem 不等式逐颗成立）→ 拿任一颗都严格 ≥ 失去它的 approach → 簇旁悬停**非不动点**、搜索
逐颗拿下。拿 A 后 A 的 approach 归 0、A 改属性使 B 的 ΔRP 略降（递减）→ B 的 approach 仍正、继续拉向 B。
（可选更紧：max over T 代 sum，上界更死；但 sum 与现成 pull_big 一致、per-gem 已够 → 推荐 sum、省一变量。）

**为什么低风险**：思路一不是新机器、是把已验证的 pull+G 结构（`big_cells` 上跑通的）套到 `T` 上；就近病证明
现成（命门④-④），只多 `β_a≤β_small` 一条。
