## 0. 一句话根因

**这颗 1攻/1防 宝石的全部价值都是【到手瞬间才兑现】(拿取奖励 G + 拿到后 boss 段损血降)；三处打分都
没有给搜索从远处『拉向道具袋』的梯度。而进袋要先穿 3 怪 2 门——一段排序键单调下行的『谷』；beam 是
近视 top-K 前沿，在够到袋底那道『崖』(G 满额兑现)之前，这条在建绕路就已被『不绕路、HP 更高』的兄弟态
帕累托压掉、剪枝出局。** 这与想法1/点1『剑盾误判』同根（回报延迟、approach 无梯度），区别仅在剑盾因 ΔRP
巨大被划进大件、还能吃到 `1/(1+dist)` 的微弱 pull，而 MT1 小宝石连这点 pull 都没有。

## 1. 区结构与几何门禁（确认玩家『3怪2门』死袋）
- 攻防宝石：`(7,3)`+a1/+d0，`(7,4)`+a0/+d1
- 门：`(2,5)`yellowKey，`(2,8)`yellowKey，`(4,3)`yellowKey，`(6,6)`yellowKey，`(6,9)`yellowKey，`(9,5)`yellowKey，`(10,9)`yellowKey
- 钥匙物品：`(1,6)`yellowKey，`(3,10)`yellowKey，`(3,11)`yellowKey，`(5,10)`yellowKey，`(8,3)`yellowKey
- 怪：`(2,4)`skeleton，`(2,7)`skeletonSoldier，`(3,1)`greenSlime，`(4,1)`redSlime，`(5,1)`greenSlime，`(7,6)`bat，`(8,6)`bluePriest，`(9,6)`bat，`(9,11)`greenSlime，`(10,10)`bat，`(11,11)`greenSlime
- 大件涌现集（数据）：`[('MT5', 11, 11), ('MT9', 9, 7)]` —— MT1 这两颗**不在**大件集，是【小宝石】。
- **几何**：宝石+钥匙+血瓶挤在右上小室，入口被【1 黄门 + 2蝙蝠1法师三怪簇 + 1 黄门】串联封死，
  是条**死袋（dead-end）**——进去拿完原路退出，不通向任何楼梯/后续区。玩家说『要穿 5 联通块(3怪2门)』属实。

## 2. 行为对照：best 路线到了 MT1 却跳过道具袋
- 甜区 best 路线**第 780 步首次到 MT1**（落点 `(2,1)` HP375 atk24/def24，手里 4 把钥匙）。
- best 终态在 MT1 这两颗宝石上：`(7,3)`**没拿**；`(7,4)`**没拿**。
- 玩家通关串：`(7,3)`**拿**；`(7,4)`**拿**。
- → 两者就在这个死袋上分道：玩家垫钥匙穿三怪拿光，搜索绕过去了。

## 3. 三处打分：哪一处都没有『拉向道具袋』的梯度
| 打分项 | 这颗小宝石得到什么 | approach（进袋途中）有没有梯度 |
|--------|--------------------|------------------------------|
| **pull_大件**（方向引导） | **恒为 0**（只对大件给 `ΔRP/(1+dist)` 梯度，小宝石没有） | **无**——根本不在引导集 |
| **拿取奖励 G** | β_small·ΔRP₀（**到手才给**满额常数） | **无**——远处=0，只在拿到瞬间跳一个崖 |
| **区势能 D 兑现** | 拿到后 atk/def↑ → boss 最短路损血↓ → 基分↑ | **无**——进袋是【远离 boss】方向，D 不降，到手才降 |

**三处全是【到手才兑现】、approach 零信号。** 搜索只有『正好扩展到袋底那一步』才会拿——而它在够到之前先被剪。

## 4. 谷-崖实证（引擎真走进道具袋）
从 best **首次到 MT1 的真快照**出发（钥匙临时充到 10、隔离钥匙荒），沿 vzone 最短损血路 `step()` 真走进
袋底（途中顺收 1 血瓶 + 防宝 `(7,4)` + 攻宝 `(7,3)`），逐步看 beam 排序键相对入口的增量：

| 步骤事件 | Δ排序键 | HP | pull | G |
|----------|--------:|---:|-----:|--:|
| 入口(MT1到达) | 0 | 375 | 0.0 | 1821270.0 |
| 开门(钥-1) | +0 | 375 | 0.0 | 1821270.0 |
| 战斗损血14 | -14 | 361 | 0.0 | 1821270.0 |
| 战斗损血24 | -38 | 337 | 0.0 | 1821270.0 |
| 战斗损血14 | -52 | 323 | 0.0 | 1821270.0 |
| 开门(钥-1) | -52 | 323 | 0.0 | 1821270.0 |
| 拾血瓶(+50hp) | -2 | 373 | 0.0 | 1821270.0 |
| ★拾防宝(+1def) | +6200 | 373 | 0.0 | 1827387.0 |
| ★拾攻宝(+1atk) | +21278 | 373 | 0.0 | 1842375.0 |

- **谷**：进袋穿 3 怪每步只掉 HP（pull 恒 0、G 恒 0、区势能不降）→ 排序键单调下行到谷底 `Δ≈-52`；
  途中血瓶也只把 HP 抠回来一点（`Δ` 仍是负的），**整条 approach 没有任何一处把分往上抬**。
- **崖**：踏上两颗宝石各跳一道崖（防宝 `+6200`、攻宝再冲到 `Δ=+21278`），G 到手才一次性兑现满额。
- **关键**：净收益 `Δ=+21278` 是**正的** → 根因**不是『奖励太小』**，坏在【顺序】：回报全堆在崖顶、谷在前。
  beam 近视 top-K，每一 wave 都拿『在建绕路』(谷里、HP 更低、G 还没兑现) 跟『不绕路兄弟』(HP 更高、
  已拿 G 相同) 比标量——绕路态在够到崖之前就被帕累托压掉、剪枝出局，**永远走不到那一步**。

## 5. 钥匙轴（次因，让绕路雪上加霜）
- 道具袋**净黄钥 = +1(袋内奖励) − 2(开 2 门进袋) = −1 把**。
- best 首达 MT1 仅 4 把钥匙、MT1 共 7 扇门 → 钥匙荒；
  beam 把有限钥匙花在【开门即兑现 HP】的血瓶门上，不肯先垫 2 把进死袋等崖顶回报。
- 钥匙在 **value_vector 是 Pareto 保护维**、但**不进标量基分 HP−D** → 净 −1 黄钥在标量里**零权重**，
  『这把钥匙将来某扇门要用』的价值近视 beam 看不见 = **想法3 已证的钥匙价值结构盲区**。

## 6. 与想法3 结构盲区对账（坐实下个 session 两个方向就是治本处）
- **延迟满足**：成本（2 钥匙 + 3 怪损血）在前、回报（2 小宝石 + 1 钥匙 + 红血）在崖顶到手才兑现 → 命中想法3 共性。
- **钥匙价值**：净 −1 黄钥只在 Pareto 轴、标量零权重 → 命中想法3『钥匙价值』盲区。
- **联通块视野**：穿 5 联通块(3怪2门)进死袋、approach 三处零梯度 → 命中『联通块/远处视野』盲区。
- 三条**全部命中** → 下个 session 的两个改法方向（联通块视野 + 钥匙价值）正对根因。
